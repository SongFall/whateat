# 实现前端API服务和优化接口逻辑

## 1. 项目结构设计

### 1.1 目录结构
```
frontend/
├── app/
│   ├── request/
│   │   └── apiClient.js        # 现有的API客户端
│   └── services/              # 新增：API服务目录
│       ├── user.js            # 用户管理API
│       ├── recipe.js          # 菜谱管理API
│       ├── article.js         # 文章管理API
│       ├── collection.js      # 收藏管理API
│       └── follow.js          # 关注管理API
└── components/
```

### 1.2 核心功能
- 基于现有`apiClient.js`实现各模块API服务
- 添加登录拦截和认证机制
- 统一错误处理
- 请求/响应拦截器优化

## 2. 实现计划

### 2.1 用户管理API (`user.js`)
- 实现用户注册、登录、获取用户信息等功能
- 添加登录状态管理
- 实现token刷新机制

### 2.2 菜谱管理API (`recipe.js`)
- 实现菜谱的增删改查
- 支持分页、搜索、筛选
- 实现菜谱图片上传功能

### 2.3 文章管理API (`article.js`)
- 实现文章的增删改查
- 支持点赞、评论、收藏功能
- 实现文章分类和标签管理

### 2.4 收藏管理API (`collection.js`)
- 实现收藏/取消收藏功能
- 获取用户收藏列表
- 支持按类型筛选

### 2.5 关注管理API (`follow.js`)
- 实现关注/取消关注功能
- 获取关注列表和粉丝列表
- 检查是否已关注

## 3. 优化接口逻辑

### 3.1 登录拦截机制
- 在请求拦截器中添加token检查
- 实现401错误处理，自动跳转到登录页
- 添加token过期自动刷新功能

### 3.2 统一错误处理
- 优化响应拦截器，统一处理API错误
- 添加错误提示机制
- 实现错误日志记录

### 3.3 请求优化
- 添加请求防抖和节流
- 实现请求缓存机制
- 支持请求取消

### 3.4 类型安全
- 添加请求和响应数据类型检查
- 实现数据验证

## 4. 实现步骤

1. 创建各模块API服务文件
2. 实现基础API功能
3. 优化请求/响应拦截器
4. 添加登录拦截和认证机制
5. 实现统一错误处理
6. 测试API服务功能

## 5. 技术要点

- 基于现有`apiClient.js`扩展
- 使用localStorage存储token
- 实现JWT认证机制
- 添加请求重试机制
- 支持TypeScript类型定义

## 6. 预期效果

- 模块化的API服务，便于维护和扩展
- 完善的登录拦截和认证机制
- 统一的错误处理和提示
- 优化的请求性能
- 类型安全的API调用

## 7. 注意事项

- 确保与后端API文档保持一致
- 实现跨域请求处理
- 添加适当的错误处理和日志记录
- 确保安全性，避免泄露敏感信息
- 优化性能，减少不必要的请求
